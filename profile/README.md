## Team Identity

우리는 단순히 LLM 데모를 만드는 팀이 아니라  
요구사항 정의 → 데이터·문서 준비 → RAG·에이전트 설계 → 프롬프트·도구·체인 구성 →  
평가·가드레일 → Streamlit 등으로 서비스화까지  
전 과정을 설계하는 엔지니어링 중심 팀입니다.

---

## Core Capabilities

- RAG·문서 기반 QA 설계 및 검색·청킹·임베딩 파이프라인 구성
- LangChain 기반 체인·에이전트·도구 연동 및 OpenAI API 활용
- 프롬프트·시스템 지침 설계와 응답 품질·할루시네이션 완화
- 평가 지표·테스트셋 기반 검증 및 가드레일·안전성 고려
- 재현 가능한 설정·버전 관리(프롬프트, 체인, 의존성)
- Git 기반 협업 (PR / Issue / Convention)
- Streamlit 기반 프로토타입·서비스 UI 구성

---

## Working Method

- Issue 기반 작업 분리
- Branch 전략 준수
- Conventional Commit 사용
- PR 리뷰 후 Merge
- 프롬프트·RAG 설정·평가 결과 및 설계 근거 문서화

---

## Repository Philosophy

각 프로젝트는 독립된 저장소로 관리하며  
다음 원칙을 따릅니다:

- 구조화된 폴더 구성
- 데이터·문서·비밀값 분리 관리
- 프롬프트·체인·모델 설정 버전 관리
- 산출물·실험 로그·평가 기록 유지
